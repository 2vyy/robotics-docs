Robotics projects like ours involve large packages, arcane documentation, and complex build systems. AI coding assistants are one of the highest-leverage tools available — they can help you generate boilerplate nodes, debug build errors, brainstorm system designs, and understand unfamiliar code fast.

But AI is a multiplier, not a replacement for understanding. Use it wisely and you'll build faster and better.

## Rules

- **Understand before you commit.** If you don't know what the AI is doing, ask it to explain. Read the diff before committing.

- **Be honest about AI help.** Say when you're taking Claude's word for something versus when you independently verified it.

- **Watch terminal commands carefully.** Be cautious with AI-suggested `git`, `apt`, `pip`, `uv`, `rm`, and `curl` commands. Check what they do before running them.
